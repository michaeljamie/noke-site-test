NOKĒ API

By utilizing the Nokē API, businesses can gain accountability and visibility by integrating their existing platforms with Nokē to avoid complicated management processes, multiple platforms, and redundancy.

INTEGRATION SIMPLIFIED

Nokē's integration solution positions Nokē as the only industrial lock manufacturer that can seamlessly plug into customers existing platforms, apps, data, and analytics, and create a network effect of systems working together.

HERE ARE THE KEY BENEFITS:

EASY TO USE

Manages only locks and keys, simply adding that data to your user management system

QUICK TO IMPLEMENT

API wrappers in most common programming languages and native mobile libraries for iOS and Android

SECURE

Two layers of 128-bit AES encryption

CUSTOMIZABLE

Provides freedom to customize all Nokē features according to individual needs

FAST

Always in active development allowing for feature requests to be quickly implemented

DOCUMENTATION

Comprehensive documentation and examples for all endpoints and sandbox environment for testing

A PLATFORM OF YOUR OWN

Nokē API not only provides seemless integration but also offers the flexibility to manage your own system

API COMMUNICATION FEATURES:

INTERNAL MANAGEMENT

Nokē only manages the encrypted keys for locks. All other user data is managed internally by client.

NOKĒ DATA ACCESS LIMITED

Mobile app, server, and all user data, event history, and company information is not accessible to Nokē.

NOKĒ SERVER ACCESS LIMITED

Communication with Nokē servers is limited to one-way requests for encrypted keys via a RESTful API and requires an API key.

BLUETOOTH

All Bluetooth communication with locks is handled through the integrated Nokē mobile library for Android and iOS.

AND WHEN IT'S TIME TO OPEN YOUR LOCK, WE'LL PROVIDE THE KEY

HERE'S HOW THE NOKĒ CORE API WORKS:

LOCK
MOBILE APP & CORE LIBRARY
NOKĒ SERVER
NOKĒ CLOUD
NOKĒ CORE API
NOKĒ CLOUD
NOKĒ SERVER
MOBILE APP & CORE LIBRARY
LOCK
Nokē lock connects to Nokē Mobile app via Bluetooth and user requests unlock
Mobile app sends lock and user data to the Nokē cloud
The cloud authenticates user and lock. If user has permission to unlock the lock, cloud sends request to the Nokē Core API
Nokē Core API generates and encrypts the unlock command using the lock's keys (stored on secure Nokē servers)
Cloud receives unlock command from API and passes it to the mobile app
Mobile app sends command to the lock and the lock opens

SAFE, SECURE, AND UNHACKABLE

Nokē API security protocols

NOKĒ API SECURITY

Utilizing an independent, end to end 128-bit AES encryption with an additional layer of 128-bit AES, the Nokē team of technology experts works hard to ensure that our system of rigorous and audited API security mechanisms and protocols are built to restrict all unauthorized access.
